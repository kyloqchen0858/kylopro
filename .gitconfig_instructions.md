# Git 配置说明

## 问题背景

仓库中的 `Kylopro-Nexus/` 文件曾以 Windows 路径格式存储（例如 `"Kylopro-Nexus\\core\\engine.py"`），
导致文件在 Linux/macOS 上无法正常识别为目录结构。

原因：从 Windows 上传文件时，`core.quotepath` 未关闭，Git 将 Windows 路径分隔符 `\` 当成转义字符处理。

## 修复方法

在你的本地仓库执行以下命令（**只需执行一次**）：

```bash
# 关闭路径转义，避免中文和特殊字符显示为 octal 编码
git config core.quotepath false

# Windows 换行符自动转换（Windows 用户必须设置）
git config core.autocrlf true
```

## 全局配置（推荐所有 Windows 用户执行）

```bash
# 全局设置（对所有 git 仓库生效）
git config --global core.quotepath false
git config --global core.autocrlf true
```

## 验证配置是否生效

执行以下命令，确认输出为 `false`：

```bash
git config core.quotepath
# 预期输出：false
```

## 上传文件时的注意事项

在 Windows 上通过 Python 脚本操作 git 路径时，需将反斜杠转为正斜杠：

```python
# 正确做法：将 Windows 反斜杠转为正斜杠
path_str = str(path).replace("\\", "/")
subprocess.run(["git", "add", path_str], ...)

# 错误做法（会导致路径乱码）
subprocess.run(["git", "add", str(path)], ...)  # Windows 路径含 \
```
